A collection of Expo brownfield apps

Needs only a minSdk 21→24 bump, a process-aware `Application`, and a one-line `androidx.lifecycle.Transformations`→`.map`/`.switchMap` migration (the AAR bumps lifecycle past the removed API); both apps launch from two overflow-menu entries ([recording](https://raw.githubusercontent.com/expo/expo-brownfield-examples/media/57-notally/recording.mp4))
- [pocket-casts-android](https://github.com/briones-agent/pocket-casts-android): Isolated approach using AARs (SDK 57 canary) — Kotlin/Compose podcast app and the first example embedding **two distinct Expo apps** in one host, published to the **GitHub Packages** Maven registry ([briones-agent/packages](https://github.com/briones-agent/packages)) rather than the static `raw.githubusercontent.com` layout (the registry rejects app tokens, so it needs a classic PAT with `read:packages`). Two full fused AARs can't co-install — AGP fused-library doesn't relocate classes, so `com.facebook.react.PackageList` / `expo.modules.ExpoModulesPackageList` / `libappmodules.so` / `assets/index.android.bundle` collide, and `ExpoReactHostFactory` is a process-wide singleton — so **App A** carries the full runtime in the main process while **App B** is a satellite AAR stripped to just its wrapper + a renamed `index.b.android.bundle`, reusing App A's runtime off the shared classpath and running in a separate `:expo_b` process for its own React host; `PocketCastsApplication` is made process-aware and both apps launch from two rows in Pocket Casts' Settings screen ([recording](https://raw.githubusercontent.com/expo/expo-brownfield-examples/media/45-pocket-casts-android/recording.mp4))
- [Trail-Sense](https://github.com/briones-agent/Trail-Sense): Isolated approach using AAR (SDK 57 canary) — fully-offline Kotlin sensors/compass app consuming the fused release AAR from local Maven; launched from a floating "Expo" button on the compass screen. The cleanest single-module, no-flavor host of the Android batch — needed only a minSdk 23→24 bump and `-Pbrownfield.fused.host-provided=com.caverock` because both expo-image and the host ship androidsvg 1.4 (duplicate-class otherwise) ([recording](https://raw.githubusercontent.com/expo/expo-brownfield-examples/media/47-trail-sense/recording.mp4))

## Integration steps

Check commits inside each repo for detailed steps, full instructions can be found in the [expo-brownfield documentation](https://docs.expo.dev/brownfield/overview/).

### Integrated

#### Monorepo

1. **Set up a yarn monorepo**: Create a `package.json` file in the root directory, then add the following to it:

   ```json
   {
     "private": true,
     "workspaces": ["exp"]
   }
   ```

2. **Create the Expo app**: Run `npx create-expo-app exp` to set up a new Expo app.

3. **Install dependencies**: Add expo to you Podfile and run `pod install`

4. **Add React Native view**: Create a new Swift file in the `NetNewsWire-iOS` target, for example `ReactNativeView.swift`, and implement a basic React Native view.

### Isolated

1. **Create the Expo app**: Run `npx create-expo-app exp` to set up a new Expo app.

2. **Install expo-brownfield**: Add expo-brownfield to your project `npx expo install expo-brownfield` and generate a XCFramework.

3. **Add React Native view**: Integrate the Expo app XCFramework into the existing iOS app.
